Sistema de Gestão de Solicitações - SGS

Projeto desenvolvido com as seguintes tecnologias:

Java
Spring Boot
HTML
CSS
JavaScript
PostgreSQL

Funcionalidades do sistema - SGS

Cadastro de solicitações
Lista as solicitações
Permissão de filtro por STATUS, CATEGORIA e PERÍODO 
Alteração de status
Detalhes da solicitação

URL para renderização das páginas Frontend application

http://localhost:8080/index.html           -> Página inicial
http://localhost:8080/detalhes.html?id=1   -> Página de detalhes
http://localhost:8080/cadastro.html        -> Página de cadastro

Endpoints da API

POST:   http://localhost:8080/sgs/cadastrar  ->  Verifica o cadastro da solicitação

{
	"solicitanteId": 2,
	"categoriaId": 3,
	"statusId": 5,
	"descricao": "Compra de 5m de fio elétrico",
	"valor": 250,
	"dataSolicitacao": "2026-05-20"
}

PUT:  http://localhost:8080/sgs/status  -> Verifica a atualização do status da solicitação

{
	"id": 8,
	"status": "LIBERADO"
}

GET:  http://localhost:8080/sgs/solicitacaoId/5   ->   mostra um único regisro de solicitação mostrando apenas o id e status de solicitação

GET:  http://localhost:8080/sgs/5   ->   mostra um único regisro de solicitação com base no id da solicitação

GET:  http://localhost:8080/sgs/listar  ->  mostra todos os registros de solicitação de forma geral

GET:  http://localhost:8080/sgs/listarStatus   ->   mostra todos os registros de status

GET:  http://localhost:8080/sgs/listarCategorias   ->   mostra todos os registros de categoria

GET:  http://localhost:8080/sgs/listarSolicitantes   ->   mostra todos os registros de solicitação apenas o id e o nome

GET:  http://localhost:8080/sgs/status/categoria/data     ->   mostra todos os registros de forma geral

GET:  http://localhost:8080/sgs/status/categoria/data?status=SOLICITADO  ->  mostra apenas os registros com base no status

GET:  http://localhost:8080/sgs/status/categoria/data?categoria=TRANSPORTE -> mostra apenas resgistros com base na categoria

GET:  http://localhost:8080/sgs/status/categoria/data?status=APROVADO&categoria=MATERIAL -> mostra apenas registros com base no status e categoria

GET:  http://localhost:8080/sgs/status/categoria/data?status=APROVADO&categoria=TI&dataInicial=2026-01-01 -> mostra apenas registros com base no status, categoria e data inicial

GET:  http://localhost:8080/sgs/status/categoria/data?status=APROVADO&categoria=TI&dataFinal=2026-02-21 -> mostra apenas registros com base no status, categoria e data final

GET:  http://localhost:8080/sgs/status/categoria/data?status=APROVADO&categoria=TI&dataInicial=2026-01-01&dataFinal=2026-05-15 -> mostra apenas os registros com base no status, categoria, data inicial e data final


Script de criação de tabelas e seus relacionamentos:

-- This script was generated by the ERD tool in pgAdmin 4.
-- Please log an issue at https://github.com/pgadmin-org/pgadmin4/issues/new/choose if you find any bugs, including reproduction steps.
BEGIN;


CREATE TABLE IF NOT EXISTS public.categoria
(
    id serial NOT NULL,
    nome character varying(100) COLLATE pg_catalog."default" NOT NULL,
    CONSTRAINT categoria_pkey PRIMARY KEY (id)
);

CREATE TABLE IF NOT EXISTS public.solicitacao
(
    id serial NOT NULL,
    solicitante_id serial NOT NULL,
    categoria_id serial NOT NULL,
    descricao character varying(200) COLLATE pg_catalog."default" NOT NULL,
    valor numeric(10, 2) NOT NULL,
    data_solicitacao date NOT NULL,
    status_id serial NOT NULL,
    CONSTRAINT solicitacao_pkey PRIMARY KEY (id)
);

CREATE TABLE IF NOT EXISTS public.solicitante
(
    id serial NOT NULL,
    nome character varying(200) COLLATE pg_catalog."default" NOT NULL,
    cpfcnpj character varying(50) COLLATE pg_catalog."default",
    CONSTRAINT solicitante_pkey PRIMARY KEY (id)
);

CREATE TABLE IF NOT EXISTS public.status
(
    id serial NOT NULL,
    descricao character varying(50) COLLATE pg_catalog."default" NOT NULL,
    CONSTRAINT status_pkey PRIMARY KEY (id)
);

ALTER TABLE IF EXISTS public.solicitacao
    ADD CONSTRAINT categoria_fk FOREIGN KEY (categoria_id)
    REFERENCES public.categoria (id) MATCH SIMPLE
    ON UPDATE NO ACTION
    ON DELETE NO ACTION
    NOT VALID;


ALTER TABLE IF EXISTS public.solicitacao
    ADD CONSTRAINT solicitante_fk FOREIGN KEY (solicitante_id)
    REFERENCES public.solicitante (id) MATCH SIMPLE
    ON UPDATE NO ACTION
    ON DELETE NO ACTION
    NOT VALID;


ALTER TABLE IF EXISTS public.solicitacao
    ADD CONSTRAINT status_fk FOREIGN KEY (status_id)
    REFERENCES public.status (id) MATCH SIMPLE
    ON UPDATE NO ACTION
    ON DELETE NO ACTION
    NOT VALID;

END;
